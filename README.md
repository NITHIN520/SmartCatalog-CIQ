## What Does It Do?

Feed it a raw product name. It returns a **Category** and **Sub-Category** вҖ” automatically, in milliseconds, using a 6-layer AI pipeline that gets smarter with every prediction.

```
INPUT   вҶ’   "Corona Extra Lager Can"

OUTPUT  вҶ’   Category     : Beer
            Sub-Category : Light Lager
            Confidence   : 94.2%
            Source       : ml
```

The more products you run through it, the smarter it gets. **Zero manual effort after setup.**

## The Self-Learning Loop

```
Run 1:  "Quilmes Cristal"       вҶ’ LLM answers (paid)  вҶ’ saved to dataset
Run 2:  "Quilmes Cristal"       вҶ’ Exact match (free)  вҶҗ remembers it вң…
Run 3:  "Quilmes Cristal Light" вҶ’ Fuzzy match (free)  вҶҗ similar name вң…
Run 4+: After retrain           вҶ’ ML handles it (free) вҶҗ fully learned вң…
```

Every cycle: **faster, cheaper, smarter.**

---

## Layer Performance at a Glance

| Layer | Speed | API Cost | Self-Learns? |
|---|---|---|---|
| Exact Match | вҡЎ Instant | Free | вң… Grows with every prediction |
| Keyword Scan | вҡЎ Instant | Free | вқҢ Hardcoded rules |
| Fuzzy Match | вҡЎ Instant | Free | вң… Grows with every prediction |
| Vector Search | рҹҡҖ Fast | Free | вң… Rebuilds on retrain |
| ML Classifier | рҹҡҖ Fast | Free | вң… Retrains periodically |
| LLM (GPT-4o-mini) | вҸұ ~1 second | Paid | вқҢ (but saves result) |

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Open .env and set your OPENAI_API_KEY
```

### 3. Seed your reference data and train the first model

```bash
python main.py seed --file my_products.json
```

Or use the built-in beverage demo data:

```bash
python main.py seed
```

### 4. Tag your products

```bash
python main.py batch products_input.csv --output products_output.csv
```

That's it. Open `products_output.csv` and your products are categorized.

### `batch` вҖ” Tag a whole CSV

```bash
python main.py batch products_input.csv --output products_output.csv

# Dry run вҖ” predict without saving to dataset
python main.py batch products_input.csv --output results.csv --no-store
```

Your input CSV needs a `product_name` or `sub_category` column:

```csv
product_name
Corona Extra Lager Can
Heineken 0.0 Non-Alcoholic Beer
Jack Daniel's Tennessee Whiskey
```

Output columns written in real-time as each row is processed:

```csv
product_name, predicted_category, predicted_sub_category,
category_confidence, sub_category_confidence,
category_source, sub_category_source, needs_review
```

The `needs_review` column is automatically flagged `YES` for low-confidence or LLM-sourced predictions.

---

### `correct` вҖ” Fix a wrong prediction

```bash
python main.py correct "BeatBox Wine Cocktail" "Ready to Drink"
```

This does three things instantly:
1. Updates the stored label in the dataset
2. Retrains the ML model
3. Rebuilds the vector index

**This is the most powerful way to improve accuracy.**

---

### `retrain` вҖ” Force the model to learn from all accumulated data

```bash
python main.py retrain
```

Run this after every ~50вҖ“100 new batch predictions to keep the ML model sharp.
Auto-triggers after 20 new predictions by default (configurable).

---

### `seed` вҖ” Load reference data and bootstrap the model

```bash
# From your own JSON
python main.py seed --file my_products.json

# Use built-in beverage demo data
python main.py seed
```

Your `my_products.json` format:

```json
[
  {"sub_category": "Craft Beer",       "category": "Beer"},
  {"sub_category": "Light Lager",      "category": "Beer"},
  {"sub_category": "Cider",            "category": "Hard Beverage"},
  {"sub_category": "Sparkling Water",  "category": "Non-alcoholic Beverage"}
]
```

### `stats` / `status` вҖ” Inspect the system

```bash
python main.py stats     # Dataset size, category breakdown, source counts
python main.py status    # Model trained? LLM available? Thresholds?
```

```
  в””в”Җ python main.py batch products_input.csv --output products_output.csv

STEP 3 В· Review results
  в””в”Җ Open products_output.csv
  в””в”Җ Look at needs_review = YES rows
  в””в”Җ Check low confidence scores

STEP 4 В· Fix wrong predictions
  в””в”Җ python main.py correct "product name" "correct category"

STEP 5 В· Retrain periodically
  в””в”Җ python main.py retrain   (every 50вҖ“100 new predictions)

STEP 6 В· Repeat from Step 2
  в””в”Җ The system gets smarter every cycle  вҷ»
```

## Configuration (`.env`)

| Variable | Default | Description |
|---|---|---|
| `OPENAI_API_KEY` | *(required)* | Your OpenAI key вҖ” only used by Layer 6 |
| `OPENAI_MODEL` | `gpt-4o-mini` | GPT model for LLM fallback |
| `ML_CONFIDENCE_THRESHOLD` | `0.75` | Min ML confidence to trust prediction |
| `FUZZY_MATCH_THRESHOLD` | `85` | Min fuzzy score (0вҖ“100) to trust string match |
| `MIN_SAMPLES_TO_RETRAIN` | `20` | New predictions needed to trigger auto-retrain |
